# 智能筛选排序功能实现计划

## 摘要

实现豆瓣榜单的智能筛选排序功能，支持按评分、年份、地区等条件筛选，按评分/热度/年份排序。

## 当前状态分析

### 现有问题

* [douban.py](file:///e:/下载/ziyong/DBAuto/app_modules/douban.py) 获取的豆瓣数据包含 `rating`（评分）但未使用

* [transfer.py](file:///e:/下载/ziyong/DBAuto/app_modules/transfer.py) 的 `build_transfer_tasks` 直接使用原始数据，没有筛选

* 搜索结果直接取第一个 `sr[0]`，没有按评分/大小等排序

* 前端界面没有筛选条件配置入口

### 数据流分析

```
豆瓣 API → get_douban_list() → 返回原始数据 → build_transfer_tasks() → 直接转存
```

    # ... 原有缓存逻辑 ...
    
    # 获取原始数据后进行处理
    result = [{
        "title": i.get("title", ""), 
        "rating": i.get("rating", {}).get("value", 0),
        "year": i.get("year", ""),
        "region": i.get("region", "")  # 如果API返回地区信息
    } for i in items if i.get("title")]
    
    # 评分筛选
    if min_rating > 0:
        result = [r for r in result if r["rating"] >= min_rating]
    
    # 年份筛选
    if year_from > 0 or year_to > 0:
        result = [r for r in result if _parse_year(r["year"], year_from, year_to)]
    
    # 排序
    if sort_by == "rating":
        result.sort(key=lambda x: x["rating"], reverse=True)

        result.sort(key=lambda x: _parse_year_int(x["year"]), reverse=True)
    
    # 限制数量
    result = result[:limit]

## 潜在风险与注意事项

### 兼容性

* 现有用户的旧配置没有 `filters` 字段，需要向后兼容

* 后端需要处理 `filters` 为 None 的情况

### 缓存影响

* 筛选参数加入缓存 key，可能导致缓存数量增加

* 需要确保缓存清理机制正常工作

### API 数据限制

* 豆瓣 API 返回的年份和地区信息可能不完整

* 需要处理数据缺失的情况
